# 組織内文書・会計管理システム 要件定義

## 概要

GitHub
Pages・Firebase・自作サーバー上のPostgreSQLを用いて、組織内で文書や会計情報の作成・承認が行えるシステムを構築する。GitHub
Pagesでは概括的なUIを提供し、詳細な情報についてはFirebase認証を経た後に自作サーバーのAPIから取得する。

### 補足事項

- 既存のユーザーリストがCSV形式で用意されており、インポート機能を実装する
- 未登録ユーザーであっても特定のドメインでログインした場合は専用画面へ遷移させる
- 組織アカウントでのFirebaseアクセスに制限があるため、当面は `gmail.com`
  ドメインのアカウントでFirebaseを使用する

---

## 環境・前提条件

- Ubuntu Server上にDockerおよびDocker Composeが導入済み
- Nginxがホスト上で動作しており、リバースプロキシとして使用中
- 他のComposeプロジェクトが既に存在し、プロジェクトごとに別ディレクトリで管理している
- 外部ネットワーク `shared_backend` は未作成
- `127.0.0.1:3001` は使用済みのため、別のポートを割り当てること

---

## プロジェクト構成

| 項目                     | 内容                      |
| ------------------------ | ------------------------- |
| 言語・フレームワーク     | Rust / Axum / SQLx        |
| データベース             | PostgreSQL                |
| リバースプロキシ         | Nginx（ホストOS上）       |
| プロジェクトディレクトリ | `~/docker/<project-name>` |

Composeプロジェクトは以下の機能単位に分割する。

- 認証サービス（Nginx `auth_request` のサブリクエスト先）
- 利用者・組織管理API + PostgreSQL
- ファイル管理API + PostgreSQL
- 会計管理API + PostgreSQL

---

## セキュリティ要件

- PostgreSQLはComposeの内部ネットワークにのみ属し、外部へ直接公開しない
- ComposeプロジェクトはUbuntu Server外部へ直接公開しない
- Firebase IDトークンの検証はNginxの `auth_request` によって認証サービスへ集約する
- APIキーは `.env` で管理し、`.env.example` も合わせて作成する

#### 組織情報テーブル（Header）

| カラム名 | 型                       | 制約                                        |
| -------- | ------------------------ | ------------------------------------------- |
| 組織ID   | SERIAL                   | PK                                          |
| 組織種別 | ENUM(permanent, special) | NOT NULL                                    |
| 親組織ID | INTEGER                  | FK → 組織情報テーブル（自己参照）, NULL許容 |
| 組織名   | VARCHAR(64)              | NOT NULL                                    |
| 対象年度 | SMALLINT                 | 特設組織のみ。常設組織はNULL                |

- 親組織がない場合は `親組織ID = NULL` の「Root」組織を親とする
- `親組織ID = NULL` となる組織はRoot以外に認めない
- CHECK制約:
  `(組織種別 = 'permanent' AND 対象年度 IS NULL) OR (組織種別 = 'special' AND 対象年度 IS NOT NULL)`

## APIの機能要件

1. **条件検索エンドポイント**:
   クエリパラメータで条件（利用者の場合、氏名・ステータス・入会日の範囲等）を指定すると、合致するレコードの一覧をJSON形式で返す
2. **登録エンドポイント**:
   指定した要素（利用者の場合、氏名・読み仮名・メールアドレス・入会日・ステータス等）をPOSTするとテーブルへ挿入する
3. 各ComposeプロジェクトおよびGoogle Apps Script（GAS）等からHTTP経由でアクセスできること
4. 全操作においてFirebase IDトークンによる認証を必須とすること
